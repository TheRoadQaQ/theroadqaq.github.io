从 Post-Training 整体的角度反思 SFT

April 12, 2026

2026   ·   research, arxiv2026     ·   Research  

现在主流推理模型的 Post-Training 基本都遵循两阶段：，先 SFT 后 RL，但是现在的 SFT 算法往往只思考自己这个阶段的效果，忽略了SFT 的终点也是后续 RL 的起点

问题不是 “SFT 或者 RL 不够强”，而是前面的 SFT 往往已经把 RL 需要的探索空间压没了，导致整体效果没有那么好

基于这个观察，我们提出了 GIFT（Gibbs Initialization with Finite Temperature）：一种把 SFT 重新理解为 RL 的最优初始化问题 的方法。

我们希望回答一个更前置的问题：

什么样的初始化策略，最适合后续 RL 去探索？

现在主流推理模型的后训练基本都遵循两阶段：

  1. SFT：先用高质量示范把模型训到“会做”
  2. RL：再通过可验证奖励让模型继续探索更优解

这个 pipeline 很常见，但是有一个根本冲突：

  • SFT 的目标是模仿专家轨迹
  • RL 的目标是探索高奖励轨迹

但是，标准 SFT 用 one-hot supervision 做交叉熵优化，会把概率质量强行压到示范 token 上，导致分布迅速塌缩。

  • 模型看起来更“像老师”，但同时也更不愿意走别的路径
  • 而 RL 恰恰需要这些没有被完全压死的替代路径来做探索

很多时候不是 RL 学不会，而是 SFT 先把可探索空间耗尽了。

标准 SFT 的问题：它其实对应 zero-temperature limit

  • 理想的 post-training 目标，应该同时考虑 任务 reward 和 base model prior
  • 如果从这个全局目标出发，最优策略其实是一个 Gibbs distribution

也就是说，专家示范不应该是对 base model 的“硬覆盖”，而应该是对原始分布的一种有温度的重加权。

在这个视角下，标准 SFT 相当于把温度压到极限：

  • temperature -> 0
  • 分布退化成几乎只剩示范答案的尖锐峰值

这也解释了为什么标准 SFT 往往会有 distributional collapse：

  • 它不是“偏向专家”
  • 而是几乎把其它可能性全部清空

从模仿学习角度这也许合理，但从 “SFT 之后还要接 RL” 的角度看，这种初始化并不理想。

GIFT

不要把 SFT 做成 one-hot imitation，而是把它做成一个 finite-temperature 的 Gibbs 初始化。

具体来说，GIFT 不是直接让模型只拟合示范 token，而是构造一个 soft target distribution：

  • 以 base model 的分布 为底
  • 再用 专家轨迹的优势/奖励信号 去重加权
  • 通过一个 inverse temperature 控制“向专家靠拢”到底有多强

于是，GIFT 的训练目标既保留了示范数据的方向，也保留了 base model 原本的结构先验。

为什么这种初始化更适合 RL

后续 RL 通常还会带一个 KL 约束，防止策略离初始化策略太远。

这意味着，初始化策略长什么样，会直接决定 RL 后面还能探索多大空间。

如果初始化已经极度塌缩：

  • RL 的搜索半径就会很小
  • 很多潜在高奖励轨迹根本采不到

而 GIFT 的目标就是：让初始化本身就和后续 RL 的最优方向一致。

不是把 SFT 和 RL 混在一起训练，而是重新定义：

  • SFT 应该服务于 RL 的全局最优
  • 而不是只服务于局部模仿误差最小

所以它解决的不是单纯的 overfitting，而是 两阶段目标不一致 这个更底层的问题。

实验结果

在两个 backbone 上评测：

  • Qwen2.5-7B
  • Llama-3.1-8B

训练数据使用 DeepMath-103k 的子集，之后统一接 GRPO 做 RL。评测既看数学推理，也看 OOD general reasoning。

数学推理：GIFT 在两个 backbone 上都最好

在五个数学 benchmark 的 RL 后平均 pass@1 上：

  • Qwen2.5-7B GIFT 达到 52.43 标准 SFT 是 48.58

  • Llama-3.1-8B GIFT 达到 35.60 标准 SFT 是 29.62

OOD 泛化：保住了 base prior，所以更稳

在 GPQA、MMLU-Redux、ARC-Challenge、MMLU-Pro 上：

  • Qwen2.5-7B GIFT 平均 64.10 标准 SFT 59.78

  • Llama-3.1-8B GIFT 平均 55.24 标准 SFT 46.97

GIFT 因为没有把分布压到 zero-temperature 的极端，所以更容易保住原有通用能力。

探索能力：pass@1 不一定领先，但 pass@k 更能拉开

我们还专门比较了 RL 前 的 pass@k scaling。

一个很有意思的现象是：

  • 标准 SFT 在 pass@1 上未必更差
  • 但随着采样数增加，它的收益增长会明显变慢

以 Qwen2.5-7B 为例：

  • RL 前，standard SFT 的 pass@1 甚至略高于 GIFT：39.79 vs. 38.92
  • 但到 pass@8，GIFT 反超到 62.81 vs. 59.01

这就是 distributional collapse 的直接证据：

  • 单样本看起来还行
  • 多采样时却没有足够多样性去持续涨点

而 GIFT 由于保留了更宽的输出分布，给后续 RL 留下了更大的搜索空间。

一个重要结论：最优温度不是 0，也不是越大越好

我们还分析了 inverse temperature 的影响，结论很清楚：

  • 太小：约束不够，离专家太远
  • 太大：又会逼近标准 SFT 的塌缩极限
  • 最优点在中间某个 finite temperature 区间

而且不同 backbone 的最佳温度不一样。

  • 问题不是 “给 SFT 稍微加点 entropy regularization 就够了”
  • 而是需要一个明确的、和全局 post-training 目标一致的温度化初始化视角

我们认为

SFT 的目标不应该只是把模型训得更像示范，而应该是为后续 RL 保留足够好的探索起点。
